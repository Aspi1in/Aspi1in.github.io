可以看出就是在当前的DexCache对象中找是不是保存的有当前DexFile对象实例的指针
调用了一个ConvertNativeToJavaArray()方法将存放DexFile对象容器的数组指针转换成一个jlongArray类型，最终返回给Java层的DexFile的openDexFile()方法中，然后再返回到

可以看到，在native层存放被加载注册的DexFile对象的容器的指针被放到一个叫mCookie的成员变量中，然后再往上回朔到DexFile类当中的loadDexFile()方法中

在DexFile类makePathElements()方法中通过调用loadDexFile()方法构造了传递进来dex文件对应的DexFile对象，并且已经加载到内存中，通过DexFile.mCookie成员可以获得其引用

构造好的载入内存的DexFile对象实例被放置到了一个Element[]类型的容器中返回给了public DexPathList(ClassLoader definingContext, String dexPath, String libraryPath, File optimizedDirectory)构造方法

最终这个存放实例化DexFile对象的容器被赋值给了DexPathList.dexElements成员当中，网上回朔到BaseDexClassLoader类的构造方法中去

构造的DexPathList实例被赋值给了BaseDexClassLoader.pathList成员变量中，在往前回朔就到了DexClassLoader类的构造方法中，自此就已经基本完成了，根据dex文件加载文件到内存的所有工作。
